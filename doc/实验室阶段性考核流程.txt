================================================================================
                    实验室阶段性考核流程（修订版）
================================================================================

一、Git 仓库协作考核
--------------------------------------------------------------------------------

本次考核不限制操作方式。

考生可自行选择：
  • Git 命令行（Terminal）
  • IDE / 编辑器图形界面（如 IDEA、VSCode、trae、Cursor 等）

重点考察的是：
  • 是否理解 Git 协作流程
  • 是否能够正确完成对应操作
  • 是否具备基础项目协作能力

1. 获取考核仓库
   进入会议后，考生需先将自己使用的代码托管平台（Gitee / GitHub）告知考官
   考官向考生发送对应的考核仓库链接

2. Fork 与本地项目准备
   • Fork 考核仓库到自己的账号下
   • 在 Fork 后的仓库中创建个人分支（建议使用姓名/昵称命名）
   • 将仓库拉取（Clone）到本地
   • 本地项目切换到个人分支进行后续操作

   本阶段主要考察：
     ◦ 是否能够完成代码拉取
     ◦ 是否能够切换并使用个人分支
     ◦ Fork 的目的是避免对原仓库造成污染

3. 云端文件修改
   • 在仓库主分支的 doc 文件夹中新建文件
   • 保存并提交修改

4. 分支同步与更新
   • 发起 Pull Request（PR）
   • 将主分支最新修改同步到个人分支
   • 将远程最新代码更新到本地项目

   本阶段主要考察：
     ◦ 是否理解 PR 与分支同步流程
     ◦ 是否能够处理远程更新
     ◦ 是否具备多人协作意识

5. 本地文件创建与代码提交
   • 在本地项目的 doc/docs 文件夹中新建文件
   • 文件中填写任意可追踪内容
   • 完成后提交修改并推送至远程仓库

   要求：
     ◦ 提交说明（Commit Message）需明确描述本次修改内容
     ◦ 代码需成功同步到远程仓库

   提交说明示例：
     docs: 新增个人考核测试文件

6. 检查远程仓库结果
   打开远程仓库页面，检查：
     ◦ 文件是否成功上传
     ◦ 提交记录是否正常显示
     ◦ 修改内容是否能够被追踪


二、项目运行考核
--------------------------------------------------------------------------------

1. 环境配置
   • 根据项目中的 .env.example 文件复制生成 .env 文件
   • 按要求修改环境配置：

   项目环境配置：
     ┌─────────────┬─────────────┐
     │   模块      │   环境      │
     ├─────────────┼─────────────┤
     │   前端      │   remote    │
     │   后端      │   local     │
     │   管理端    │   local     │
     └─────────────┴─────────────┘

2. 前端项目运行
   • 进入前端项目目录
   • 安装依赖并启动项目（方式不限，可使用命令行或 IDE 内置工具）
   • 详细启动说明请查看项目根目录的 README.md 文件（见「启动步骤 > 2. 选择启动方式 > 方式一/二」章节）
   • 成功启动后，根据考官要求完成页面表单填写

   本阶段主要考察：
     ◦ 前端环境配置能力
     ◦ 项目启动能力
     ◦ 基础问题排查能力

3. 后端与管理运行
   • 根据 README 文档启动后端服务
   • 在本地启动项目前后端服务
   • 根据考官要求完成页面操作

   本阶段主要考察：
     ◦ Java/Maven 环境配置
     ◦ 后端服务运行能力
     ◦ 前后端联调能力

4. 数据库连接检查
   • 管理方向需连接云端数据库
   • 检查页面提交的数据是否成功入库

   重点检查：
     ◦ 数据库连接是否正常
     ◦ 数据是否成功写入
     ◦ 接口调用是否正常


三、考核重点
--------------------------------------------------------------------------------

本次考核主要评估以下能力：
  • Git 基础操作能力
  • 分支协作与 PR 流程理解
  • 项目环境配置能力
  • 前后端项目运行能力
  • 数据库联调能力
  • 基础问题排查能力
  • 独立完成开发任务能力


四、注意事项
--------------------------------------------------------------------------------

  • 本次考核不涉及代码编写，主要考察 Git 操作熟练度
  • 全程需独立完成操作
  • 操作方式不限，但需保证流程正确
  • 提交信息需规范、清晰
  • 禁止直接在主分支进行开发
  • 所有修改需保证可追踪、可同步、可运行
